Backup Management

The recommended way to handle backups is via the included daily_backup.sh script, which can be run on a cron job. However, this system lets you manually manage backups.




Restoring from one of the files below will wipe the database completely and re-load it from the file. You cannot undo this process.

Book List Exports

These are lists of books only -- for example if you want to move a bunch of books from one uBiblio library to another. Or if you just want a CSV file of your books that you can open with other appliations.


Restoring from one of these files will not wipe the current database. It will add all the books from the file to the current database. Generally it is a good idea to backup the database before attempting this.
